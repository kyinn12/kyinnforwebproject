Chapter 9: Creating JavaScript Objects - Summary

1. What is an Object?

  • In programming, an **Object** is the fundamental unit for storing and processing data.
  • In JavaScript, an object is a collection of related **information** and **actions** bundled together.
  • **Types of Objects**:
    • **Built-in Objects**: Pre-defined objects (e.g., `Date`, `Math`).
    • **Document Object Model (DOM)**: Web documents, including elements like images, links, etc.
    • **Browser Object Model (BOM)**: Information used by the web browser (e.g., `window`, `location`).
    • **User-defined Objects**: Objects created by the user as needed.

2. Creating User-defined Objects

User-defined objects are created using the **Object Literal** method, consisting of **properties** (key-value pairs).

  • **Syntax**: `objectName = { key1: value1, key2: value2, ... }`.
  • **Key**: Must be a string, number, or Symbol. Keys are separated by a colon (`:`).
  • **Accessing Properties**:
    • **Dot Notation**: `object.key` (Key is used without quotes).
    • **Bracket Notation**: `object["key"]` (Key must be enclosed in quotes).
  • **Modifying/Adding/Deleting**: Properties can be modified or added using assignment (`=`), and deleted using the `delete` keyword.
    book1.pages = 50;
    delete book2.pages;
  • **Object Nesting**: Objects can contain other objects (or nested objects). Access nested properties using sequential dot notation (e.g., `student.score.history`).

3. Object Methods and `this`

  • **Method**: A property of an object whose value is a **function**, defining the object's behavior.
    Traditional Syntax: `methodName: function() { ... }`
    ES6 Shorthand: `methodName() { ... }`
  • **`this` Keyword**: Inside an object's method, `this` is a reserved keyword that refers to the **current object**. It is used to access other properties within the same object (e.g., `this.title`).

4. Object Copying

  • **Primitive Data Copy**: When copying primitive data types (e.g., numbers), the **value** is copied. Changing the copied variable does **not** affect the original.
  • **Object Copy**: When copying objects, the **memory address (reference)** is copied. Changing a property in the copied object **will change the original object's value** because both point to the same data location.

5. Constructor Functions and Classes

When multiple objects share the same properties and methods, you can create a **template** (constructor) to easily create new instances.

  • **Constructor Function**: A template function used to create objects. By convention, the name starts with a capital letter.
    Inside the function, properties are defined using `this`.
    Instance creation: `let book1 = new Book("Title", 648);`
  • **Class (ES6 Syntactic Sugar)**: A modern, simpler syntax (syntactic sugar) for creating constructor functions.
    The **`constructor(parameters)`** method initializes properties using `this`.
    Instance creation: `let book1 = new BookC("Title", 648);`

6. Accessing Keys and Values

  • **`for...in` Loop**: Used to iterate through and retrieve only the **keys** (or both keys and values) of an object.
    for (key in object) { console.log(key); }
  • **`Object.keys(object)`**: Returns an array containing only the **keys** of the object.
  • **`Object.values(object)`**: Returns an array containing only the **values** of the object.
  • **`Object.entries(object)`**: Returns an array containing an array of `[key, value]` pairs for each property.

7. Prototypes and Inheritance

  • **Prototype**: The original form from which a JavaScript object is created. Every object inherits properties and methods from its prototype.
    The `[[Prototype]]` property (or `__proto__`) tells you which object an instance was derived from.
  • **Prototype Method**: A method can be defined outside the constructor function using `prototype`. This saves memory as the method is shared by all instances.
    Book.prototype.finish = function() { ... };
  • **Inheritance (Class)**: Classes use the **`extends`** keyword to inherit from a parent class.
    The **`super(properties)`** function must be called inside the child class's `constructor()` to reuse the parent class's properties.
    class TextbookC extends BookC { super(title, price); this.major = major; }
