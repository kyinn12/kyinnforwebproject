Chapter 2: Variables and Data Types - Summary

1. Input and Output Methods (입력과 출력)

  • **`alert(content)`**: Displays an **alert box** with a message or variable value.
  • **`confirm(content)`**: Displays a **confirmation box** with "OK" and "Cancel" buttons. Returns `true` if "OK" is clicked, and `false` if "Cancel" is clicked.
  • **`prompt(content, defaultValue)`**: Displays a **prompt box** where the user can input a value. If the user clicks "Cancel" (or doesn't input a value), it returns `null`.
  • **`console.log(content)`**: Displays content in the **console window**, often used for debugging.
  • **`document.write(content)`**: Displays content directly on the **web browser screen**. (Note: This is used temporarily before studying the DOM.)

2. Variables and Declaration (변수와 선언)

  • A **Variable** is a name (like a label) assigned to a memory space (like a basket) used to store data values within a program.
  • **Declaration**: Variables are declared using a keyword followed by the variable name.
    • **`let`**: Used to declare a variable whose value **can be changed (reassigned)** later.
    • **`const` (Constant)**: Used to declare a constant whose value **cannot be changed** (reassigned) after initialization. Constants are useful for values that are logically fixed (e.g., the current year or a DOM element).
    • **`var`**: The older keyword (pre-ES6), which allows both re-declaration and reassignment.
  • **Assignment**: A value or expression is stored in a variable using the assignment operator (`=`). Declaration and assignment can be done simultaneously.

3. Variable Naming Rules (변수 이름 규칙)

  • Cannot start with a **number** or contain **spaces**.
  • Is **case-sensitive** (e.g., `current`, `Current`, and `CURRENT` are all different).
  • Cannot use **reserved keywords** (e.g., `let`).
  • **Common Conventions** (for multi-word variables):
    • **Snake Case**: Use underscores (`_`) (e.g., `current_year`).
    • **Camel Case**: Capitalize the first letter of each word except the first (e.g., `currentYear`).

4. Data Types (자료형)

JavaScript data types are categorized into **Primitive** (single value) and **Object** (all other types).

  • **`number` (숫자형)**: Includes both integers and floating-point (real) numbers.
  • **`string` (문자열)**: Data enclosed in single (`' '`) or double (`" "`) quotes.
    • **Escape Characters**: Special characters (like quotes or new lines) are represented using a backslash (`\`) (e.g., `\'`, `\n`).
    • **Template Literal**: An ES6 format using **backticks** (`` ` ``) that allows embedding variables or expressions directly inside the string using `${}`.
  • **`boolean` (논리형)**: Represents either **`true`** or **`false`**.
    • **Falsy Values**: Certain values are treated as `false` in logical contexts: `0`, `""` (empty string), `NaN`, `undefined`, and `null`. All other values are considered **Truthy**.
  • **`undefined`**: The initial type and value of a variable that has been declared but not yet assigned a value.
  • **`null`**: The type and value representing an intentionally invalid or empty value.
  • **`array` (배열)**: Stores multiple values in a single variable, enclosed in square brackets (`[]`) and separated by commas.
    • **Index**: Values are accessed by their **index**, starting from **0**.
    • The size of the array is given by the **`.length`** property.
  • **`Symbol`**: A new primitive type (ES6+) that guarantees **uniqueness**. Used mainly as a key in object properties to prevent naming conflicts.
  • **Checking Type**: The **`typeof(value)`** function returns the data type of a value or variable.

5. Type Conversion (자료형 변환)

JavaScript is a loosely typed language, and data types can change during execution.

  • **Automatic (Implicit) Conversion**: Occurs during operations. For example, the subtraction operator (`-`) automatically converts a string to a number. The addition operator (`+`) acts as a **concatenation operator** if one operand is a string.
  • **Explicit Conversion**: Use built-in functions to control the conversion:
    • **To Number**: `Number(value)`, `parseInt(value)` (for integer), `parseFloat(value)` (for float).
    • **To String**: `value.toString()`, `String(value)` (can convert `null` and `undefined`).
    • **To Boolean**: `Boolean(value)`.
    • **Fixed Decimal**: `number.toFixed(digits)` fixes the number of decimal places for a floating-point result.
