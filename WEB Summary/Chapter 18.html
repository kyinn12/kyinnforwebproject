Chapter 18: Utilizing Public APIs - Summary

1. Understanding Public APIs (공개 API)

    [cite_start]
  • A **Public API (Open API)** is an API made publicly available for anyone to use[cite: 2].
    • [cite_start]
  • Examples include map APIs or data APIs provided by major portal sites[cite: 13].

1.1. Public API Operation Principle

[cite_start]

Since the APIs are on a server, you must apply for use and receive an authentication key to access and use them[cite: 4].

    [cite_start]
  • The **Developer** requests an **Authentication Key (인증키 요청)** [cite: 11] [cite_start]and receives **Authentication Key Approval (인증키 승인)**[cite: 14].
    • [cite_start]
  • This key allows the developer to access the API through a legitimate path and use its functions or properties[cite: 5].
    • [cite_start]
  • The API facilitates communication between the developer's application and the **Server Data** by handling **Requests (요청)** and **Responses (응답)**[cite: 10, 12].

    [cite_start]
  1. Sign up on the Kakao Developer page[cite: 38].
    2. [cite_start]
  2. **Add an application (애플리케이션 추가하기)**[cite: 39].
    3. [cite_start]
  3. **Confirm App Keys**: Check keys like the JavaScript Key and REST API Key[cite: 77, 78].
    4. [cite_start]
  4. **Configure Platform**: Select 'Web Platform Registration' (Web 플랫폼 등록) [cite: 84] [cite_start]and enter the site domain (e.g., the local host address http://127.0.0.1)[cite: 90, 97].
    5. [cite_start]
  5. Go to the Kakao Map API site and click **[Web]**[cite: 106].
    6. [cite_start]
  6. Reference the various sample sources in the **'Sample'** menu for implementation[cite: 117].
      [cite_start]
    • **Getting the API**: Use a <script> tag with the appropriate source URL[cite: 151, 152].
      • [cite_start]
    • **Area for the Map**: Define a <div> element with an ID to hold the map[cite: 142, 147].
      • [cite_start]
    • **Displaying the Map**: Use JavaScript to set coordinates (e.g., new kakao.maps.LatLng(33.450701, 126.570667)) and create the map object[cite: 161, 164, 168].

3. Utilizing Open API from the Public Data Portal (공공데이터포털)

3.1. About the Public Data Portal

    [cite_start]
  • It is a site run by the government that continuously provides new public data[cite: 181, 182].
    • [cite_start]
  • **Public Data**: Refers to data created by public institutions or official information opened to the public[cite: 183].
    • [cite_start]
  • Anyone can use the data for free after applying[cite: 184].
    • [cite_start]
  • Data is provided as **File Data**, **Open API**, and **Standard Data Set**[cite: 187].
      [cite_start]
    • **Open API**: Data is standardized and shared for programming[cite: 189]. [cite_start]It remains **up-to-date** because information can be modified in real-time on the server[cite: 190].

3.2. Open API Application Process

    [cite_start]
  1. Search for the required information on the Public Data Portal[cite: 193].
    2. [cite_start]
  2. Click the **[Open API] tab** in the search results[cite: 194].
    3. [cite_start]
  3. After reviewing the details, click **[Apply for Utilization] (활용 신청)**[cite: 195].
    4. [cite_start]
  4. An **'General Authentication Key'** is issued for the applied API[cite: 212].
    5. [cite_start]
  5. Developers must review the **'Reference Document' (참고문서)** which details API usage[cite: 213].

3.3. Example: Fetching Campground Data and Displaying it on a Map

Using the Korea Tourism Organization's GoCamping Information Lookup Service (REST type):

  1. **Check Request Method**: Refer to the API manual for the server request method. 
    // Example URL to request JSON format:
http://api.visitkorea.or.kr/openapi/service/rest/GoCamping/basedList?ServiceKey=ServiceKey&numOfRows=10&pageNo=1&MobileOS=ETC&MobileApp=TestApp&_type=json
    * [cite_start]Add &_type=json to the URL to request a JSON response[cite: 272].
    2. [cite_start]
  2. **Fetch Data**: Add the App Key (앱키) [cite: 281] [cite_start]to the URL and use fetch to retrieve the data[cite: 277, 283].
    3. [cite_start]
  3. **Access Content**: Analyze the returned JSON object structure (e.g., in the console) to find the path to the actual data[cite: 290]. 
    // Accessing the actual list of items:
[cite_start]const data = json.response.body.items.item // Stores the actual content from the object. [cite: 336]
  4. **Identify Necessary Fields**: To display on a map, the necessary fields are:
      [cite_start]
    • Campground Name: `facltNm` [cite: 357]
      • [cite_start]
    • Latitude value: `mapY` [cite: 357]
      • [cite_start]
    • Longitude value: `mapX` [cite: 357]
    5. [cite_start]
  5. **Display on Map**: Use the retrieved data in conjunction with the Kakao Map API to mark the campground locations[cite: 360].
