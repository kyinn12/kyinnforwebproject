Chapter 12: HTTP Communication and JSON - Summary

1. HTTP Communication (HTTP 통신)

To exchange data between a client (web browser) and a server, a pre-arranged set of rules called a **protocol** is necessary. The web uses the **HTTP** (HyperText Transfer Protocol) protocol.

  • **HTTP Request**: The client requests data from the server. This includes the **Request Header** (containing system info, browser type, etc.) and optionally the **Request Body**.
  • **HTTP Response**: The server sends data back to the client. This includes the **Response Header** (status, message time) and the **Response Body** (the actual content like HTML, images).
  • **HTTPS**: The secure version of HTTP, recommended for all websites by modern browsers.

2. HTTP Request Methods: GET vs. POST

  • **GET Method**: Used when requesting data. The request message is appended to the URL (Visible in the address bar). No separate request body is used.
    • **Use Case**: Safe for requests where data disclosure (like search keywords) is not a concern.
  • **POST Method**: Used when sending data that should not be exposed (like login credentials). The content is sent in the **Request Body**, hidden from the URL.
    • **Use Case**: Mandatory for sensitive operations like logins, form submissions, and resource creation.

3. Response Status Codes (응답 상태)

The server sends a numerical **Status Code** in the response header indicating the result of the request.

  • **2XX (Success)**: Request was accepted or successfully completed.
    • `200 OK`: Successful transmission from server to client.
  • **4XX (Client Error)**: The client made a bad request.
    • `400 Bad Request`: General failure of the request.
    • `404 Not Found`: The requested document/resource could not be located.
    • `403 Forbidden`: Access denied (even if authenticated).
  • **5XX (Server Error)**: The server encountered an internal error.
    • `500 Internal Server Error`: General server-side error.

4. Data Exchange Format: JSON

  • **JSON (JavaScript Object Notation)** is the preferred data exchange format on the web (over XML) due to its speed and simplicity.
  • **Features**: Text-based, fast transmission speed, language-independent, and easy to convert to/from JavaScript objects.
  • **JSON Format**: Composed of key-value pairs enclosed in curly braces `{}`.
    • **Key**: Must always be enclosed in **double quotes** (e.g., `"name"`).
    • **Value**: Can be a number, string (must be in double quotes), boolean (`true`/`false`), `null`, or an array/nested JSON string. **Functions are not allowed as values.**
    • Multiple JSON strings can be stored together as an **Array** (`[]`).

5. JSON and JavaScript Conversion

  • **Stringify (Serialization)**: Converting a JavaScript **object** into a JSON **string** to be stored or sent to a server.
    JSON.stringify(object)
  • **Parse (Parsing)**: Converting a JSON **string** (retrieved from a server) back into a JavaScript **object** to be used in programming.
    JSON.parse(JSON_string)

6. Asynchronous Communication (AJAX)

  • **Asynchronous Communication**: The process of bringing in only parts of a web document without reloading the entire page. This is handled by **AJAX** (Asynchronous JavaScript and XML).
  • **`XMLHttpRequest` (XHR)**: The object used for HTTP communication (requesting data from a server and receiving the response) in JavaScript.
    • **Instance Creation**: `let xhr = new XMLHttpRequest();`
    • **Setup**: `xhr.open(method, url, isAsync);` (method is GET/POST, URL is the resource, `isAsync` is typically `true`).
    • **Send**: `xhr.send(content);` (content is optional, usually for POST requests).
  • **XHR Response Properties**:
    • **`xhr.readyState`**: Indicates the connection status (0 to 4). `4` means the data processing is complete and ready to use.
    • **`xhr.status`**: The HTTP status code (e.g., `200`).
    • **`xhr.responseText`**: The response data stored as a **string**.
    • **`onreadystatechange`**: An event handler executed every time `readyState` changes. Used to check if `readyState === 4` and `status === 200` to process the received data.

7. Error Handling (예외 처리)

**Exception Handling** is anticipating and managing issues that could stop a program (e.g., syntax errors, user canceling a prompt).

  • **`try...catch...finally`**:
    • **`try`**: Contains the code to execute.
    • **`catch (error)`**: Executes if an exception occurs in the `try` block. Used to display the error (e.g., `console.error(err)`).
    • **`finally`**: Executes after the `try` and `catch` blocks, regardless of whether an exception occurred.
  • **`throw`**: Allows the user to manually create an exception and specify an error message (e.g., `throw "User name missing.";` or `throw new Error("...");`).
