Chapter 3: Operators and Control Statements - Summary

1. Operators (연산자)

Operators are symbols that perform an operation on **operands** (the values being manipulated).

  • **Arithmetic Operators (산술 연산자)**: Perform mathematical calculations.
    • `+`, `-`, `*`, `/`: Standard addition, subtraction, multiplication, and division.
    • `%` (Modulus): Returns the **remainder** of a division.
    • `++` (Increment): Increases the operand by 1. `--` (Decrement): Decreases the operand by 1.
      • **Prefix (`++a`)**: Performs the operation **before** the value is used in the expression.
      • **Postfix (`a++`)**: Performs the operation **after** the value is used in the expression.
  • **Assignment Operators (할당 연산자)**: Assign the result of the right-side expression to the left-side variable.
    • `=` (Assignment)
    • Compound operators: `+=`, `-=`, `*=`, `/=`, `%=` (e.g., `y += x` is equivalent to `y = y + x`).
  • **Concatenation Operator (연결 연산자)**: The `+` symbol is used to join strings together. **If either operand is a string, JavaScript treats `+` as concatenation.**
  • **Comparison Operators (비교 연산자)**: Compare two values and return a boolean result (`true` or `false`).
    • `==` (Equal), `!=` (Not Equal): Compares values, **automatically converting data types** if necessary (e.g., `3 == "3"` is `true`).
    • `===` (Strict Equal), `!==` (Strict Not Equal): Compares values and checks if the **data types are identical**. (e.g., `3 === "3"` is `false`).
  • **Logical Operators (논리 연산자)**: Combine and manipulate boolean values, often used in complex conditions.
    • `||` (OR): Returns `true` if **at least one** operand is `true`.
    • `&&` (AND): Returns `true` only if **all** operands are `true`.
    • `!` (NOT): Inverts the boolean value of the operand.
  • **Operator Precedence**: Determines the order in which operators are executed in an expression (Unary > Arithmetic > Comparison > Logical > Assignment).

2. Conditional Statements (조건문)

Statements that allow the program to make decisions based on whether a condition is true or false.

  • **`if` Statement**: Executes a block of code only if the condition inside the parentheses is `true`.
    if (condition) { /* code to execute if true */ }
  • **`if...else` Statement**: Executes the `if` block if the condition is `true`, or executes the `else` block if the condition is `false`.
    if (condition) { /* true code */ } else { /* false code */ }
  • **`if...else if...else`**: Links multiple conditions sequentially. The first condition to evaluate to `true` is executed.
  • **Conditional Operator (조건 연산자)**: A shorthand for simple `if...else` structures.
    (condition) ? true_result : false_result;
  • **`switch` Statement**: Used when there are many possible conditions to check against a single variable/expression.
    • The variable is checked against each `case` value.
    • **`break`**: Must be used after the commands in a `case` to exit the `switch` statement immediately.
    • **`default`**: Code that runs if none of the `case` values match.
  • **Short-Circuit Evaluation (단축 평가)**: In `&&` and `||` operations, if the result can be determined after checking the first condition, the rest of the expression is skipped, making processing faster.

3. Looping Statements (반복문)

Statements used to execute the same code multiple times, reducing repetition and shortening code.

  • **`for` Loop**: The most common loop, typically used when the number of repetitions is known.
    for (initialization; condition; increment) { /* commands */ }
  • **`while` Loop**: Continues to execute as long as the condition is `true`. It checks the condition **before** executing the code.
    while (condition) { /* commands */ }
  • **`do...while` Loop**: Executes the code **at least once**, then checks the condition. It continues to execute as long as the condition is `true`.
    do { /* commands */ } while (condition)
  • **`for...of` Loop**: Used to iterate through the **values** of an iterable object (like strings or arrays).
    for (variable of object) { /* commands */ }
  • **`for...in` Loop**: Used to iterate through the **keys** (or property names) of an object.
  • **`break` Statement**: Immediately exits the current loop (or `switch` statement), regardless of the condition.
  • **`continue` Statement**: Stops the current iteration of the loop and skips to the next iteration.
