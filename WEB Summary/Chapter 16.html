Chapter 16: Canvas Animation - Summary

1. Drawing Shapes Using Objects

  • To draw and manage multiple complex graphics (like circles) easily, it's efficient to define the graphic element as a **JavaScript object** (often using a constructor function or class).
  • The object (e.g., `Circle`) should contain all necessary properties, such as center coordinates (`this.x`, `this.y`), size (`this.radius`), and style (`this.color`).
  • The object should include a **method** (e.g., `this.draw()`) that contains the Canvas API code necessary to render itself on the screen.
  • **Instance Creation**: Once the object is defined, you can create many instances with different properties (e.g., `const circleOne = new Circle(x, y, r, c);`).
  • **Efficiency**: For drawing many similar elements, it is more efficient to use a loop to create an array of instances and then iterate through the array, calling the `draw()` method on each instance.

2. Creating Simple Animations

  • The simplest animation involves moving an element by constantly changing its coordinates (`x` and `y`).
  • To manage movement, properties are added to the object to control direction/speed (e.g., `this.dx`, `this.dy`).
  • **Animation Loop**: For continuous movement, the function that updates the coordinates and redraws the element must be run repeatedly. The standard method for smooth animation in the browser is `window.requestAnimationFrame()`.
    requestAnimationFrame(move); (The `move` function calls itself recursively).

3. Smooth Movement (Frame Redrawing)

To create the illusion of smooth movement, the canvas must be completely redrawn in every frame:

  1. **Clear Canvas**: Use `ctx.clearRect(0, 0, canvas.width, canvas.height)` at the beginning of the animation loop (`move()` or `update()`) to erase everything from the previous frame.
  2. **Update Coordinates**: Change the element's position (e.g., `circle.x += circle.dx;`).
  3. **Redraw Element**: Call the `draw()` method at the new coordinates.

4. Boundary Detection and Bouncing

To keep an element within the canvas boundary and make it "bounce," you must check its position relative to the canvas edges and reverse its direction upon contact.

  • **Collision Logic**: Check if the element's edge (center + radius) exceeds the canvas boundary (width/height) or falls below 0. 
    if (circle.x + circle.radius > canvas.width || circle.x - circle.radius < 0) {
    circle.dx = -circle.dx; // Reverse horizontal direction
}
  • **Bouncing Animation**: For complex animations involving multiple objects (like bouncing balls), a main `update()` function handles the clearing and iterates through an array of objects, calling an `animate()` method on each object instance.
  • **`animate()` Method**: This method is part of the object definition (`Circle`) and handles two things:
    1. Updating the object's position based on `this.dx` and `this.dy`.
    2. Checking for boundary collisions and reversing `this.dx` or `this.dy` if necessary.
    3. Calling `this.draw()` to render the object at its new position.
