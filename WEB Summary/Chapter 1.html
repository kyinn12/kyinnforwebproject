Chapter 1: Getting Started with JavaScript - Summary

1. Core Characteristics (자바스크립트의 특징)

  • JavaScript works on **all web browsers**.
  • Execution results can be immediately confirmed in the web browser.
  • It can be used for **full-stack web development** and creating various types of programs.
  • It utilizes various public (open) APIs, libraries, and frameworks.

2. Standardization and Evolution

  • **JavaScript vs. Java**: They are completely different languages. JavaScript was originally called **LiveScript** and was renamed to capitalize on Java's popularity at the time.
  • **ECMAScript (ES)**: To standardize the competing script languages like JavaScript and JScript, ECMA International adopted the JavaScript-based standard named ECMAScript. JavaScript is the most famous language that adheres to the ECMAScript standard.
  • **ES6 (ECMAScript 2015)**: The language was expanded to handle complex concepts following the rise of HTML5, as JavaScript needed to evolve beyond simple dynamic effects. ECMAScript continues to be updated annually.

3. JavaScript in Web Development

  • **Client-Server Model**: Websites function by exchanging information between a **client** (the user's device/browser) and a **server**.
  • **Front-end Development**: Refers to everything the user sees and interacts with, including content creation, design, and responding to user actions.
  • JavaScript's role in front-end development is increasingly powerful, leading to the emergence of popular frameworks and libraries:
    • **React**: Facebook library for creating user interfaces.
    • **Vue.js**: Framework for creating web applications and user interfaces.
    • **Angular**: Google framework, suitable for large-scale web development.
  • Major websites like Google, YouTube, and Facebook use **JavaScript** for client-side processing.

4. Development Environment

  • **Console Window**: The web browser contains a **JavaScript engine** that can execute code. Simple code can be run directly in the browser's Console window (accessible via shortcuts like `Ctrl/Option + Shift + J`).
  • **Code Editor**: **Visual Studio Code (VS Code)** is recommended for editing code.
  • **VS Code Setup**:
    • Install the D2Coding font, which supports Korean, and configure it in VS Code settings.
    • Install **Live Server** extension to automatically reload the browser when code is modified/saved.
    • Install **Prettier** to automatically format source code neatly.

5. Writing JavaScript Code

JavaScript code can be included in an HTML document in three ways:

  • **Inline Script**: Written directly inside an HTML tag attribute, often for simple commands like showing an alert.
    <button onclick="alert('Message!')">Click!</button>
  • **Internal Script**: Written between `<script>` and `</script>` tags anywhere in the HTML document, though typically placed before the `</body>` tag.
  • **External Script (Recommended)**: JavaScript source code is saved in a separate **`.js`** file and linked to the HTML document using the `<script>` tag's `src` attribute.
    • **Advantages**: Allows code reuse across multiple documents and separates markup from script logic, making maintenance easier.
    • Example: <script src="js/changeBg.js"></script>.

6. How Browsers Process JavaScript

  1. The browser recognizes the document as a web document (`<!DOCTYPE html>`) and starts interpreting the HTML structure according to the standard.
  2. It analyzes the order and inclusion relationships of all HTML tags.
  3. It processes external stylesheets and internal styles and displays them on the screen.
  4. When the browser encounters a **`<script>` tag**, it immediately passes the script source to the **JavaScript Interpreter**.
  5. The JavaScript is executed.
