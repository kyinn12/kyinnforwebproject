Chapter 6: Events and Event Handlers - Summary

1. Understanding Events

  • An **Event** is any action performed by a user (e.g., clicking a button) or the web browser (e.g., loading a page).
  • Events must occur within the web document's area.
  • **Key Event Categories**:
    • **Document Loading**: `load`, `unload`, `error`, `resize`, `scroll`.
    • **Mouse**: `click`, `dblclick`, `mousedown`, `mouseup`, `mousemove`, `mouseover`, `mouseout`.
    • **Keyboard**: `keydown` (while pressing), `keypress` (when pressed), `keyup` (when released).
    • **Form**: `focus`, `blur`, `change`, `submit`, `reset`.

2. Event Handling Methods (이벤트 처리기)

An **Event Handler** (or **Event Processor**) is the code that specifies the action to be performed when an event occurs.

  1. **Inline in HTML Tag**: Directly link a function in the HTML tag using `onEventName`.
    <button onclick="alert('Click!')">
    Note: This method is less flexible as HTML and JavaScript are mixed.
  2. **Directly Connect Function to Web Element**: Connect a function using the element's property (`.onEventName`).
    element.onclick = function() { /* commands */ }
    Note: Do not use parentheses `()` after the function name when assigning an existing function (e.g., `button.onclick = changeBackground;`).
  3. **`addEventListener()` Method (Recommended)**: The standard and most flexible way to register an event handler on a DOM element.
    element.addEventListener(event, function, useCapture);
    • **`event`**: The event type name **without** "on" (e.g., `click`, not `onclick`).
    • **`function`**: The function to run (can be a named function or an anonymous/arrow function).
    • **`useCapture`**: Boolean (optional). `true` for capturing, `false` (default) for bubbling.

3. The `event` Object

When an event occurs, an `event` object is automatically created, containing information about the event.

  • **Key Properties**:
    • **`e.target`**: Returns the element where the event **first occurred**.
    • **`e.currentTarget`**: Returns the element where the event is **currently being processed** during propagation.
    • **Mouse Position**: `e.pageX`, `e.pageY` (relative to the document).
    • **Keyboard**: `e.code` (key code), `e.key` (key name).
    • **Modifier Keys**: `e.shiftKey`, `e.ctrlKey`, `e.altKey` (return boolean).
  • **Key Method**:
    • **`e.preventDefault()`**: Cancels the default browser action for that event (if possible).

4. Event Propagation (이벤트 전파)

Event Propagation describes how an event, once triggered on an element, is passed to its parent elements.

  • **Event Bubbling (버블링)**: The default method. The event starts at the element where it occurred and propagates upward to the parent elements (like bubbles rising).
  • **Event Capturing (캡처링)**: The event starts at the top-most element (``) and propagates downward to the element where it occurred.
    To enable capturing, the third option in `addEventListener()` must be set to `true`.
    element.addEventListener("click", function, **true**);

5. Practical Example: Creating a Carousel

  • A carousel is a sliding show for images or content.
  • Images are stored in an **array** in JavaScript, separate from the HTML markup.
  • The background image of the container is set using:
    container.style.backgroundImage = `url(images/${pics[i]})`;
  • The click event on the arrows uses **`e.target.id`** (e.g., `"left"` or `"right"`) to determine the action, and **conditional logic** handles image looping (going from the last image back to the first, and vice versa).
