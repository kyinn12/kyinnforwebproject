Chapter 4: Functions and Scope - Summary

1. The Role of Functions (함수)

  • Functions are essential for **analyzing and solving problems** in programming.
  • A large problem is broken down into smaller issues, and the smallest units are written as functions.
  • Functions allow for **code reuse** and prevent the continuous repetition of the same commands for different data inputs.

2. Function Declaration, Execution, and Return

  • **Declaration**: Use the reserved keyword `function`, followed by the function name and curly braces containing the commands.
    function functionName() { /* commands */ }
  • **Execution (Calling)**: Execute the function by writing the function name followed by parentheses `()`.
    functionName();
  • **Return Value**: The `return` keyword is used to pass the result of the function's execution back to the point where it was called.
    return sum;

3. Parameters and Arguments (매개변수와 인수)

  • These allow functions to accept external values, making them reusable for various tasks.
  • **Parameter (매개변수)**: Variables declared in the parentheses when defining the function to receive values from outside. Parameters are local to the function they are declared in.
  • **Argument (인수)**: The actual values or variables passed into the function when it is executed.
  • **Default Parameters**: ES6 allows assigning a default value to a parameter during definition, which is used if an argument is missing during execution.

4. Variable Scope (스코프)

  • **Scope** is the effective range of a declared variable.
  • **Function Scope (`var`)**: Variables declared with `var` have function scope.
    • A variable declared within a function is a **local variable**.
    • A variable declared outside any function is a **global variable** and can be used throughout the program.
  • **Block Scope (`let`, `const`)**: Variables declared with `let` or `const` have block scope.
    • A **block** is any area enclosed by `{` and `}`.
    • These variables are only valid within the block where they are declared (Block Variables).
  • **Hoisting (호이스팅)**: `var` variables are hoisted, meaning the declaration is recognized internally before the code is executed, which can lead to unexpected results if the variable is used before it is assigned. **`let` and `const` do not have this issue**.

5. Modern Function Syntax

  • **Function Expression (함수 표현식)**: Assigning an anonymous function to a variable, treating the function like a value.
    let sum = function(a, b) { return a + b }
  • **Immediately Invoked Function Expression (IIFE)**: A function executed immediately upon declaration, useful for single-use functions.
    (function() { /* commands */ })();
  • **Arrow Function (화살표 함수)**: ES6 syntax using `=>` for a simpler function expression.
    • Can omit `()` if there is only one parameter.
    • Can omit `{}` and `return` if the function body is a single `return` statement.
      let sum = (a, b) => a + b;

6. Callback and Rest Parameters

  • **Callback Function (콜백 함수)**: A function passed as an argument to another function, to be executed later.
    bttn.addEventListener("click", display); // display is the callback
  • **Spread Syntax (전개 구문)**: Uses `...` to unpack values from iterable items (arrays, strings, objects).
  • **Rest Parameters (나머지 매개변수)**: Uses `...` in a function declaration to collect an indefinite number of arguments into an array.
    function addNum(...numbers) { /* commands */ }

7. Timer Functions (타이머 함수)

Functions that schedule the execution of a callback function based on time, measured in milliseconds (1s = 1000ms).

  • **`setInterval(callback, time)`**: Executes the callback repeatedly at set intervals.
  • **`clearInterval(timer)`**: Stops a function that was set to repeat using `setInterval()`.
  • **`setTimeout(callback, time)`**: Executes the callback only once after the specified time delay.

8. Debugging (디버깅)

  • **Debugging** is the process of finding and fixing errors when a program's output differs from the expected result.
  • The Browser Developer Tools include a 'Sources' tab where you can set a **breakpoint** (or stop point) by clicking the line number to pause execution.
  • This allows you to check how the source code executes step-by-step and inspect variable values (`n`, `sum`, `i`) at that point.
